基础

概念

  • 可扩展标记语言。

    1、Extensible 标签都是自定义的.

    2、Markup Language 标记语言

  • 功能

    存储数据

    ​ 1、配置文件

    ​ 2、在网络中传输

  • XML和html的区别

    w3c 万维网联盟。浏览器的恶性竞争导致html没有唯一规范。XML要求严格的语法。XML现主用于配置文件。

    ​ 1、XML的标签都是自定义的。而html标签都是预定义的。

    ​ 2、XML的语法严格。html语法松散

    ​ 3、XML是存储数据的。html是展示数据的

4、属性值必须使用双引号(单双都可)引起来

6、XML标签区分大小写

入门案例

组成部分

1、文档声明

​ 格式：< ? xml 属性列表 ?>

​ 属性列表

​ version ： 版本号 必须的属性！

​ encoding： 编码方式 告知解析引擎当前文档使用的字符集，默认：ISO-8859-1

​ standalone： 是否独立 取值 yes==不依赖其他文件 no==依赖其他文件。可以不设置

2、指定

  • 结合css的
    • < ?xml-stylesheet type=”text/css” href=”a.css” ?>

3、标签

  • 标签名称自定义的–规则
    • 名称可以包含字母、数字以及其他的字符
    • 名称不能以数字或者标签符号开始
    • 名称不能以字母 XML 或者 xml Xml等等 开始
    • 名称不能包含空格

4、属性

​ id属性值唯一

5、文本

  • CDATA区：在该区域中的数据会被原样展示
    • 格式 <![CDATA [要展示的数据] ]>

6、约束

说明文档：规定XML文档的书写规则

  • 约束文档

  • 分类

    • DTD 一种简单的约束技术

      • 引入dtd文档到xml文档中

        • 内部dtd：将约束规则定义到XML文档中

        • 外部dtd：将约束的规则定义在外部的dtd文件中

          • 本地：< ! DOCTYPE 根标签名 SYSTEM “dtd文件的位置” >
          • 网络：< ! DOCTYPE 根标签名 PUBLIC “dtd文件名字” “dtd文件的位置URL” >

    • Schema 一种复杂的约束技术

解析

操作XML文档，将文档中的数据读取到内存中

  • 操作XML文档

    • 1、解析 (读取) ：将文档中的数据读取到内存中
    • 2、写入：将内存中的数据保存到XML文档中。持久化的存储。

  • 解析XML的方式

    • 1、DOM：将标记语言文档一次性加载进内存，在内存中会形成一颗DOM树。
      • 优点：操作方便，可以文档进行CRUD的所有操作
      • 缺点：占内存
    • 2、SAX：逐行读取，基于事件驱动的。
      • 优点：不占内存。适用于较小设备 比如手机
      • 缺点：只能读取，不能增删改。

    服务器端一般使用DOM。移动端一般使用SAX

  • XML常见的解析器
    • 1、JAXP：sun公司提供的解析器，支持DOM和SAX两种思想
    • 2、DOM4J：一款非常优秀的解析器。
    • 3、Jsoup：一款Java的html解析器，可直接解析某个URL地址、html文本内容。它提供一套非常省力的API，可通过DOM、CSS以及类似于jquery的操作方法来读取和操作数据。
    • 4、PULL：android操作系统内置的解析器，SAX方式的。

  • Jsoup

    • 操作步骤

      • 1、导入jar包
      • 2、获取document对象
      • 3、获取对应的标签element对象
      • 4、获取数据

    • 对象使用

      • Jsoup：工具类，可以解析html或XML文档，返回document
        • parse：解析html或XML文档。返回document
      • document：文档对象。代表内存中的dom树
        • 获取element对象
      • elements：元素element对象的集合，可以当成arraylist< element>
      • element：元素对象
        • 1、获取子元素对象
        • 2、获取属性值
        • 3、获取文本内容
      • Node：节点对象
        • 是document和element的父类

  • 快捷查询方式

    • 1、selector
      • 选择器
    • 2、XPath
      • xpath 即为XML路径语言，它是一种用来确定XML文档中某部分位置的语言
      • 使用jsoup的

web服务器

概述

  • 软件架构

    • C/S
      • 客户端/服务器端
    • B/S
      • 浏览器/服务器端

  • 资源分类

    • 静态资源
      • 所有用户访问后，得到的结构都是一样的。称为静态资源
        • html、CSS、Javascript
    • 动态资源
      • 每个用户访问相同资源后，得到的结果可能不一样。称为动态资源。
        • servlet/JSP、php、asp

    浏览器中有 [静态资源解析引擎]

    动态资源先转换为静态资源，然后被浏览器解析

    请求/响应

  • 网络通信三要素

    • IP
      • 电子设备在网络中的唯一标识
    • 端口
      • 应用程序在计算机中的唯一表示。0～65536
    • 传输协议
      • 规定了数据传输的规则
        • TCP 安全协议，三次握手
        • UDP 不安全协议。速度快。
